# Docs Guidance

The `docs/` folder is the shared project memory for maintainers and agents.

Use it for:

* implementation references
* architecture notes
* scope and product intent
* brand identity and design direction
* expected behavior
* known problems, caveats, and unstable areas

Update the relevant docs when you:

* create a new tool or feature
* significantly change existing behavior
* discover an important mismatch, caveat, or broken workflow

Keep docs practical and easy to scan. Prefer updating the most relevant existing file before creating a new one.
